# Agents Anywhere Web

React + Vite frontend for Agents Anywhere.

## Layout

```text
src/
  components/   Shared UI components and icons
  lib/          API client, session storage, runtime helpers, theme helpers
  pages/        Auth, dashboard, device, workspace, session detail, admin views
test/           Frontend-focused utility tests
vite.config.ts  Vite config and backend proxy setup
```

## Run

Install dependencies:

```bash
yarn install
```

Start the Vite dev server:

```bash
yarn dev
```

The dev server listens on `127.0.0.1:5173`. API and WebSocket routes are proxied
to `http://127.0.0.1:8000` by default.

Override the backend target when needed:

```bash
AGENTS_ANYWHERE_API=http://127.0.0.1:8000 yarn dev
```

## Backend URL Behavior

In dev mode, Vite exposes the configured proxy target to the UI for display
purposes. In production builds, the frontend is served by the backend, so the UI
falls back to `window.location.origin`.

Connector pairing commands generated by the UI use the current browser origin as
their `--server-url`.

## Build

```bash
yarn build
```

The production build writes static assets to `dist/`. The backend can serve
that directory when `AGENT_SERVER_STATIC_DIR` points at it.

## Verify

```bash
yarn build
```

`yarn build` runs TypeScript project build before the Vite production build.
